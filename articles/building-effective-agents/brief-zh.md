# 构建有效智能体 — 简要提炼

## 核心观点

Anthropic 基于与数十个团队的协作经验，提出构建 LLM 智能体系统（agentic systems）的核心原则：**优先选择简单、可组合的模式，而非复杂框架**。最成功的实现往往不依赖专门库，而是用少量代码直接调用 LLM API。

## 关键论据

文章将智能体系统分为两类：**工作流（Workflows）**——LLM 和工具沿预定义代码路径编排；**智能体（Agents）**——LLM 动态掌控自身流程和工具使用。文章依次介绍了五种从简到繁的模式：

1. **提示链（Prompt Chaining）**：将任务拆为顺序步骤，用程序检查点确保流程正确，以延迟换精度。
2. **路由（Routing）**：分类输入并导向专门处理，实现关注点分离。
3. **并行化（Parallelization）**：分段并行执行或多次投票，提升速度与可信度。
4. **编排者-工作者（Orchestrator-Workers）**：中央 LLM 动态分解任务、分配给工作 LLM、综合结果，适用于子任务不可预判的场景。
5. **评估者-优化者（Evaluator-Optimizer）**：一个 LLM 生成、另一个评估反馈，循环迭代直到达标。

智能体则是在上述基础上更进一步——LLM 自主规划、使用工具、从环境反馈中修正，适用于开放性、步骤不可预知的问题，但代价是更高成本和复合错误风险。

## 实践要点

- 仅当简单方案不足时才增加复杂度；单次 LLM 调用加检索和示例往往已够。
- 框架降低上手门槛，但抽象层可能掩盖提示与响应，影响调试。
- 工具设计至关重要：格式应贴近模型自然见过的文本、避免格式开销、投入与 HCI 同等精力打造 ACI（智能体-计算机接口）。

## 结论

成功不在于构建最复杂的系统，而在于构建**适合需求的系统**。三条核心原则：保持设计简洁、明确展示规划步骤、精心打磨工具文档与测试。先用简单提示起步，充分评估后再引入多步智能体系统。
