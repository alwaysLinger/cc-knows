# 通过 MCP 执行代码：构建更高效的智能体

## 核心观点

MCP（Model Context Protocol，模型上下文协议）作为连接 AI 智能体与外部系统的开放标准，自2024年11月发布以来已被广泛采用。然而，当智能体连接的工具数量增长到成百上千时，传统直接工具调用方式面临严重的效率瓶颈：工具定义占用大量上下文窗口，中间结果反复经过模型处理，导致延迟增加、成本上升。

**代码执行（Code Execution）** 是解决这一问题的关键方案。其核心思路是：不再将所有工具定义直接加载到上下文中让模型逐一调用，而是让智能体通过编写代码来与 MCP 服务器交互。具体实现方式是将 MCP 工具呈现为文件系统中的代码 API，智能体按需加载所需工具定义，在执行环境中处理数据后再将精简结果返回模型。

## 主要论据

- **工具定义过载**：连接数千工具时，模型需先处理数十万 token 的定义才能开始工作
- **中间结果浪费**：如会议记录等大文档在工具调用链中可能多次流经模型，2小时会议记录就约5万 token
- **代码执行显著降本**：通过按需加载，token 用量可从15万降至2千，节省98.7%
- **隐私保护**：中间数据默认留在执行环境中，仅显式输出的内容进入模型上下文；还可对敏感数据进行自动令牌化（tokenization）处理
- **状态持久化与技能积累**：智能体可将中间结果写入文件、保存可复用函数，逐步构建高阶能力

## 结论

代码执行将软件工程中成熟的模式——按需加载、数据过滤、状态管理——应用于智能体，使其用熟悉的编程结构与 MCP 高效交互。但需权衡安全沙箱等基础设施开销。Cloudflare 的"Code Mode"也得出了类似结论：LLM 擅长编写代码，应利用这一优势构建更高效的智能体。
