# Claude Code Auto Mode：更安全地跳过权限审批

Anthropic 工程团队发现，Claude Code 用户对 93% 的权限提示点击了"批准"，这种审批疲劳促使他们构建了 auto mode——一种介于手动审批与 `--dangerously-skip-permissions` 之间的新模式。

**核心机制**：auto mode 采用双层防御。输入层通过提示注入探针（prompt-injection probe）扫描工具输出，在可疑内容进入上下文前注入警告；输出层由转录分类器（transcript classifier）评估每个操作是否与用户意图一致，充当人工审批的替代。分类器运行在 Sonnet 4.6 上，采用两阶段设计：第一阶段为快速单 token 过滤（偏向拦截），第二阶段仅对被标记的操作进行链式推理以减少误报。

**威胁模型**：auto mode 针对四类风险——过度积极行为（overeager behavior）、诚实错误、提示注入和模型错位。分类器主要针对前两类，但其防御逻辑对所有四类均适用。

**分层权限决策**：第一层为内置安全工具白名单与用户设置；第二层允许项目内文件操作直接通过；第三层才是转录分类器，仅处理有实际风险的操作。

**评估结果**：在 10,000 条真实流量中误报率仅 0.4%，但对 52 个真实过度积极行为的漏报率为 17%。漏报主因是分类器难以判断会话中的"同意信号"是否真正覆盖当前操作的破坏范围。Anthropic 坦诚表示：auto mode 适用于替代无防护模式，而非替代谨慎的人工审查。

**关键设计决策**：剥离助手文本和工具结果以防止代理说服分类器做出错误判断；两阶段分类平衡效率与召回；拒绝后继续运行而非中断会话，使误报可存活。

auto mode 不追求完美，而是在零防护与人工审查之间提供实质性的安全提升。
