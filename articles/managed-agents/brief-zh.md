# Scaling Managed Agents: Decoupling the Brain from the Hands — 简要提炼

## 核心观点

Anthropic 提出了 Managed Agents 的架构设计理念：**将智能体的"大脑"（模型与控制循环）与"双手"（沙箱与工具）解耦**，通过稳定的接口抽象来应对未来不可预知的变化。

## 关键论据

1. **线束假设会过时**：控制循环（harness）编码了关于模型能力的假设，但这些假设随模型升级而失效。例如，为 Sonnet 4.5 设计的上下文重置机制在 Opus 4.5 上成为冗余——模型已自行解决了"上下文焦虑"问题。

2. **耦合导致运维困境**：早期将所有组件放入单一容器的方案，使容器成为不可丢失的"宠物"（pet）。容器故障意味着会话丢失，调试受限，且无法连接客户的 VPC 环境。

3. **解耦带来三重收益**：
   - **可恢复性**：沙箱成为"牲畜"（cattle），故障后可重建；线束崩溃后通过会话日志恢复
   - **安全性**：凭据与沙箱隔离，提示注入攻击无法获取令牌
   - **性能**：p50 TTFT 降低约 60%，p95 降低超过 90%

4. **会话即外部上下文**：会话日志作为独立于上下文窗口的持久存储，通过 `getEvents()` 接口支持灵活检索，将上下文存储与上下文管理分离。

## 结论

Managed Agents 本质上是一个**元线束（meta-harness）**——对接口形状有主见，对接口背后的实现无偏见。这种设计借鉴操作系统的思路：用稳定的抽象（进程、文件）容纳未来未知的程序，使系统随模型智能的增长而持续演进。
