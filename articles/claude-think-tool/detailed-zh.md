# "Think"工具详细摘要

## 什么是"think"工具

"think"工具为 Claude 提供了一个额外的思考步骤，在生成响应过程中拥有专门的思考空间。它与"扩展思考"（extended thinking）是不同概念：扩展思考是在生成响应前的深度规划，而 think 工具是在响应已开始后，用于停下来审慎处理新获取的外部信息（如工具调用结果）。think 工具的推理不如扩展思考全面，更聚焦于模型发现的新信息。

扩展思考适用于简单工具调用、非顺序调用、编码/数学/物理等无需工具的场景；think 工具则更适合复杂工具调用、长链工具输出分析、策略密集型环境和序列决策。

实现方式非常简单，只需定义一个工具：接收 `thought` 字符串，不获取新信息也不改变环境，仅将思考记录到日志。

## τ-Bench 基准测试表现

τ-Bench 测试模型在真实客服场景中使用工具的能力，包括与模拟用户对话、遵循复杂策略指南、操作数据库。评估指标为 pass^k，衡量 k 次独立试验全部成功的概率，强调一致性和可靠性。

测试了四种配置：基线、仅扩展思考、仅 think 工具、think 工具+优化提示词。

**航空领域**：think 工具+优化提示词 pass¹ 达 0.570（基线 0.370，相对提升 54%）；仅 think 工具 0.404；扩展思考 0.412。在 pass^k（k=1~5）各维度上，优化配置均大幅领先。

**零售领域**：think 工具（无额外提示）pass¹ 达 0.812（基线 0.783，扩展思考 0.770）。零售策略较简单，Claude 仅凭思考空间即可改善。

**关键洞察**：困难领域需配合优化提示词，简单领域仅提供工具即可；think 工具在多次试验中保持改善，说明有助于处理边缘情况。

## 优化提示词的作用

航空领域的优化提示词在系统提示中给 Claude 示例，展示如何使用 think 工具：列出适用规则、检查信息完整性、验证行动合规性、迭代检查工具结果。提示词包含具体的决策树示例（如取消航班、预订含行李的机票），展示如何分解复杂指令为可操作步骤。将复杂指导放在系统提示中比放在工具描述中更有效。

## SWE-Bench 表现

在 SWE-bench 评估中，适配的 think 工具定义鼓励 Claude 在发现 bug 来源后头脑风暴修复方案、在收到测试结果后思考修复方法。实验（n=30 有 think 工具，n=144 无）显示平均提升 1.6%（Welch's t 检验显著，d=1.47），助力 Claude 3.7 Sonnet 取得 0.623 的当时最优成绩。

## 何时使用与何时不使用

**适用场景**：(1) 需要仔细分析工具输出再行动时；(2) 需遵循详细策略并验证合规时；(3) 序列决策中每步建立在前步之上、错误代价高昂时。

**不适用场景**：(1) 非顺序工具调用（单次或并行调用）；(2) 简单指令跟随（约束少、默认行为已足够）。代价是增加提示长度和输出 token。

## 实施最佳实践

1. **策略性提示与领域示例**：提供清晰的使用说明和领域定制示例，包括期望的推理细节程度、复杂指令分解方法、常见场景决策树、信息完整性检查方式。
2. **将复杂指导放在系统提示中**：长/复杂的 think 工具说明放在系统提示比工具描述更有效，帮助模型将思考过程整合到整体行为中。

## 入门建议

从挑战性场景开始测试（Claude 当前在策略合规或长链推理中挣扎的用例），添加定制化的 think 工具定义和系统提示中的使用说明，观察实际使用方式并优化提示。该工具副作用极小——除非 Claude 决定使用，否则不改变外部行为，也不干扰现有工具或工作流。
