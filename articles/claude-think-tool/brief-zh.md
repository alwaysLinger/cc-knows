# "Think"工具：让 Claude 在复杂工具调用中停下来思考

## 核心观点

Anthropic 提出了一种"think"工具（think tool），让 Claude 在执行复杂工具调用链时拥有专门的思考空间。与"扩展思考"（extended thinking）不同——后者是在生成响应前的深度规划，think 工具是在已开始生成响应后，提供一个中间步骤来审慎处理新获取的外部信息。

## 主要论据

- **τ-Bench 基准测试**：在航空客服领域，think 工具配合优化提示词达到 pass¹ 得分 0.570，相比基线 0.370 提升 54%；在零售领域，单独使用 think 工具即达 0.812，优于基线 0.783。
- **SWE-Bench**：加入 think 工具后，Claude 3.7 Sonnet 取得了 0.623 的当时最优成绩，独立实验显示平均提升 1.6%（统计显著）。
- **关键发现**：在困难领域，think 工具必须配合领域特定的优化提示才能发挥最大效果；在简单领域，仅提供 think 工具即可改善表现。

## 适用场景

think 工具最适合三类场景：需要仔细分析工具输出结果的场景、需要遵循复杂策略规则的环境、以及每一步都建立在前一步基础上的序列决策场景。不适用于非顺序的工具调用或简单指令跟随。

## 结论

think 工具实现简单、副作用小，能显著提升 Claude 在策略合规和长链工具调用推理中的表现，但并非万能方案，应针对具体场景选择使用。
