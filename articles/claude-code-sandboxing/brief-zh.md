# Claude Code 沙箱机制：更安全、更自主的 AI 编程

Anthropic 于 2025 年 10 月发布 Claude Code 的沙箱（Sandboxing）功能，核心目标是在提升安全性的同时减少权限提示弹窗。内部数据显示，沙箱可将权限提示减少 84%。

## 核心思路

传统 Claude Code 采用权限审批模型——每次修改文件或执行命令都需用户确认，导致"审批疲劳"（approval fatigue），反而降低安全性。沙箱的思路是：预设安全边界，让 Claude 在边界内自由操作，而非逐次请求许可。

## 两大隔离机制

沙箱建立在操作系统级别，提供两层隔离：

1. **文件系统隔离**：Claude 只能读写指定目录，防止篡改敏感系统文件。
2. **网络隔离**：Claude 只能连接已批准的服务器，防止数据泄露或下载恶意软件。

两者缺一不可——没有网络隔离，被入侵的 Agent 可外泄 SSH 密钥；没有文件系统隔离，Agent 可逃逸沙箱获取网络访问。

## 两大新功能

1. **沙箱化 Bash 工具**：基于 Linux bubblewrap 和 macOS seatbelt 实现，对命令及其子进程均生效。超出边界的访问会即时通知用户。该功能已[开源](https://github.com/anthropic-experimental/sandbox-runtime)。
2. **Web 版 Claude Code**：在云端隔离沙箱中运行，敏感凭证（如 git 凭证、签名密钥）始终不在沙箱内，通过自定义代理服务安全处理 git 操作。

## 结论

沙箱机制使即使遭受提示注入攻击，Claude Code 也无法窃取密钥或连接攻击者服务器，从根本上提升了 AI 编程助手的可信度与自主性。
