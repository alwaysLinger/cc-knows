# Claude Code 沙箱机制：更安全、更自主的 AI 编程（详细摘要）

## 文章背景

2025 年 10 月 20 日，Anthropic 发布文章介绍 Claude Code 的全新沙箱功能。Claude Code 让 AI 能够读写代码、编辑文件、运行命令，但广泛的权限带来了风险，尤其是提示注入（prompt injection）场景。为此 Anthropic 推出两个基于沙箱的新功能，在提升安全性的同时让 Claude 更自主地运行。内部数据显示，沙箱可安全减少 84% 的权限提示。

## 用户安全：权限模型的困境

Claude Code 默认采用权限审批模型——只读模式，修改文件或执行命令前需用户确认。少数安全命令（如 echo、cat）自动放行，但大多数操作仍需显式批准。

持续点击"批准"不仅拖慢开发流程，还会导致"审批疲劳"（approval fatigue）——用户不再仔细审查所批准的内容，反而降低了安全性。

## 沙箱：更安全、更自主的方案

沙箱的核心思路是创建预设边界，让 Claude 在边界内自由操作，而非逐次请求许可。其安全基础建立在操作系统级别，实现两层隔离：

1. **文件系统隔离**：Claude 只能访问和修改指定目录，防止被提示注入的 Claude 修改敏感系统文件。
2. **网络隔离**：Claude 只能连接已批准的服务器，防止被提示注入的 Claude 泄露敏感信息或下载恶意软件。

文章强调，有效的沙箱必须同时具备两种隔离。没有网络隔离，被入侵的 Agent 可外泄 SSH 密钥等敏感文件；没有文件系统隔离，Agent 可轻易逃逸沙箱获取网络访问权。两种技术结合才能提供真正安全且高效的 Agent 体验。

## 新功能一：沙箱化 Bash 工具

Anthropic 推出新的沙箱运行时（beta 研究预览），允许开发者精确定义 Agent 可访问的目录和网络主机，无需启动和管理容器的开销。该运行时可用于对任意进程、Agent 和 MCP 服务器进行沙箱化，并已作为开源研究预览发布。

在 Claude Code 中，该运行时用于沙箱化 bash 工具，使 Claude 可在设定边界内自主安全地执行命令。若 Claude 尝试访问沙箱外的资源，会即时通知用户，由用户决定是否放行。

技术实现基于操作系统级原语：
- Linux 上使用 bubblewrap
- macOS 上使用 seatbelt

这些限制覆盖 Claude Code 的直接操作以及命令衍生的所有脚本、程序和子进程。

文件系统隔离方面，允许读写当前工作目录，阻止修改外部文件。网络隔离方面，仅允许通过 Unix 域套接字连接到沙箱外的代理服务器，代理服务器对可连接域名实施限制，并处理新请求域名的用户确认。还支持自定义代理以对出站流量执行任意规则。

关键安全保证：即使提示注入成功，沙箱也能完全隔离，被入侵的 Claude Code 无法窃取 SSH 密钥或连接攻击者服务器。

启动方式：在 Claude Code 中运行 `/sandbox`，参考官方文档配置。

## 新功能二：Web 版 Claude Code

Web 版 Claude Code 在云端隔离沙箱中运行，每个会话在独立沙箱中执行，Claude 对其服务器拥有完全访问权限，同时保持安全。

核心设计原则：敏感凭证（如 git 凭证、签名密钥）始终不在沙箱内。即使沙箱内代码被入侵，用户也不会受到进一步危害。

Git 操作通过自定义代理服务透明处理：沙箱内的 git 客户端使用自定义的范围限定凭证（scoped credential）向代理认证；代理验证凭证和 git 操作内容（如确保只推送到配置的分支），然后附加正确的认证令牌发送到 GitHub。

## 快速开始

1. 在 Claude Code 中运行 `/sandbox`，参考文档配置沙箱。
2. 访问 claude.com/code 体验 Web 版 Claude Code。
3. 构建自有 Agent 的团队可使用开源沙箱代码集成。
