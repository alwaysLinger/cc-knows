# 近期 Claude Code 质量问题事后总结（详细）

**原文**: [An update on recent Claude Code quality reports](https://www.anthropic.com/engineering/april-23-postmortem)（2026年4月23日）

## 概述

过去一个月，Anthropic 调查了用户关于 Claude 质量下降的报告，追溯至三个分别影响 Claude Code、Claude Agent SDK 和 Claude Cowork 的独立变更。API 未受影响。三个问题均已于4月20日（v2.1.116）修复。Anthropic 强调从不有意降低模型质量，并确认 API 和推理层未受影响。

## 问题一：默认推理努力等级从 high 降至 medium

**背景**：2月发布 Opus 4.6 时，默认推理努力（reasoning effort）设为 `high`。此后收到反馈，高努力模式下模型偶尔思考过久，导致 UI 冻结、延迟和 token 用量过高。

**变更**：3月4日，团队将默认努力等级从 `high` 改为 `medium`。内部评估显示 medium 在多数任务上智能略低但延迟显著减少，且消除了长尾延迟问题，还能最大化用户用量限额。团队在产品中添加了说明对话框来解释此变更。

**问题**：上线后用户反馈 Claude Code 变得不够智能。团队进行了多次设计迭代以提高努力等级设置的可见性（启动通知、内联选择器、恢复 ultrathink），但大多数用户仍保持 medium 默认值。

**修复**：4月7日回退。现默认 Opus 4.7 为 `xhigh`，其他模型为 `high`。影响范围：Sonnet 4.6 和 Opus 4.6。

## 问题二：缓存优化持续丢弃推理历史

**背景**：Claude 在推理过程中产生的思考内容通常保留在会话历史中，供后续轮次参考。Anthropic 使用提示缓存（prompt caching）降低连续 API 调用的成本和延迟，但闲置一段时间后缓存会被驱逐。

**变更**：3月26日，团队设计了优化方案——当会话闲置超过一小时后，清除旧的思考部分以减少未缓存 token 数量，降低恢复会话的成本。实现使用了 `clear_thinking_20251015` API 头及 `keep:1` 参数。

**Bug**：本应只清除一次思考历史，实际代码在会话剩余的每一轮都执行清除。这导致 Claude 逐渐丢失推理上下文——如果用户在工具调用中途发送后续消息，当前轮的推理也会被丢弃。表现为健忘、重复和不合理的工具选择。由于持续丢弃思考块导致缓存未命中，还引发了用户报告的用量限额加速消耗。

**排查困难**：两个不相关的实验干扰了问题复现——一个内部服务端消息队列实验，以及一个思考显示方式的变更恰好抑制了该 bug 在多数 CLI 会话中的表现。该 bug 处于 Claude Code 上下文管理、Anthropic API 和扩展思考的交汇点，通过了多轮人工和自动化代码审查、单元测试、端到端测试及内部试用（dogfooding）。从发现到确认根因耗时超过一周。

**有意思的发现**：团队用 Opus 4.7 对问题 PR 进行回溯代码审查，在提供完整代码仓库上下文时 Opus 4.7 能发现该 bug，而 Opus 4.6 不能。团队正在为代码审查工具增加更多仓库上下文支持。

**修复**：4月10日（v2.1.101）。影响范围：Sonnet 4.6 和 Opus 4.6。

## 问题三：系统提示词变更降低编码质量

**背景**：Opus 4.7 相较前代有明显的冗长倾向，虽然在难题上表现更好，但产出更多 token。发布前数周，团队开始针对新模型调优 Claude Code。

**变更**：4月16日，在系统提示词中添加了字数限制指令——"工具调用间的文本保持在25词以内，最终回复保持在100词以内（除非任务需要更多细节）"。经数周内部测试和评估无回归后上线。

**问题**：调查期间进行更广泛的消融实验（ablation），发现该指令导致 Opus 4.6 和 4.7 编码评估下降约3%。

**修复**：4月20日立即回退。影响范围：Sonnet 4.6、Opus 4.6 和 Opus 4.7。

## 后续改进措施

1. **内部使用公开版本**：确保更多内部员工使用与公众完全相同的 Claude Code 构建，而非测试新功能的内部版本。
2. **改进代码审查工具**：增强内部 Code Review 工具，并将改进版本发布给客户。
3. **严格管控系统提示词变更**：每次系统提示词变更都运行针对各模型的完整评估套件；持续进行消融实验以理解每行提示词的影响；构建了新的工具使提示词变更更易审查和审计；在 CLAUDE.md 中增加指引，确保模型特定变更仅作用于目标模型。
4. **增加缓冲期和渐进发布**：对于可能牺牲智能的变更，增加缓冲期（soak period）、更广泛的评估套件和渐进式发布。
5. **改善沟通**：创建 @ClaudeDevs 社交账号，深度解释产品决策及理由；在 GitHub 上同步更新。
6. **重置用量限额**：4月23日为所有订阅用户重置用量限额。
