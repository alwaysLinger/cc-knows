# 近期 Claude Code 质量问题事后总结（简要）

**原文**: [An update on recent Claude Code quality reports](https://www.anthropic.com/engineering/april-23-postmortem)（2026年4月23日）

Anthropic 追踪到近期用户反馈的 Claude Code 质量下降问题，根源是三个独立变更，均已于4月20日修复。

**问题一：默认推理努力等级下调。** 3月4日，为解决高努力模式下偶发的极长延迟（UI冻结感），团队将默认推理努力（reasoning effort）从 `high` 降至 `medium`。虽然延迟改善，但用户普遍反映 Claude 变"笨"了。4月7日回退，现默认为 Opus 4.7 使用 `xhigh`，其他模型使用 `high`。

**问题二：缓存优化导致思考历史丢失。** 3月26日，为减少闲置会话恢复时的延迟和成本，团队引入清除旧思考块的机制，但实现存在 bug——本应只清除一次，实际每轮都清除，导致 Claude 逐渐丢失推理上下文，表现为健忘、重复和不合理的工具调用，还引发了缓存未命中导致用量加速消耗。4月10日修复。

**问题三：系统提示词变更损害编码质量。** 4月16日，为抑制 Opus 4.7 的冗长倾向，在系统提示词中加入了字数限制指令，与其它提示变更叠加后导致编码质量下降约3%。4月20日回退。

三个问题在不同时间影响不同用户群体，叠加后看起来像是广泛的、不一致的质量退化。Anthropic 表示将加强内部使用公开版本、扩展评估覆盖、严格管控系统提示词变更，并为所有订阅用户重置用量限额。
