# 为智能体编写有效工具 — 简要提炼

本文由 Anthropic 发布，核心探讨如何为 LLM 智能体（Agent）设计和优化工具，以提升其在真实任务中的表现。

**核心观点：** 工具是确定性系统与非确定性智能体之间的新型契约。传统软件面向开发者编写函数和 API，而为智能体设计工具则需要重新思考——目标是扩大智能体能有效解决问题的策略空间。

**方法论：** 作者提出"原型→评估→协作优化"的迭代流程。首先快速搭建工具原型并本地测试；然后构建全面的评估体系，生成大量贴近真实场景的评估任务（避免过于简单的"沙盒"环境）；最后让 Claude 等智能体分析评估结果并自动改进工具。实验表明，Claude 优化的工具甚至超越了人类专家手工编写的版本。

**五大原则：**

1. **选对工具**：工具不在多而在精。优先实现面向高影响力工作流的工具，而非简单包装 API。例如用 `schedule_event` 替代 `list_users` + `list_events` + `create_event` 的组合。

2. **命名空间**：用前缀/后缀对工具分组（如 `asana_projects_search`），帮助智能体在海量工具中准确定位，减少误调用。

3. **返回有意义的上下文**：优先返回自然语言标识而非技术 ID（用 `name` 而非 `uuid`），可显著降低幻觉率。可通过 `response_format` 参数让智能体控制返回详略。

4. **Token 效率**：实现分页、过滤、截断，并设置合理默认值。错误信息应清晰可操作，而非晦涩的错误码。

5. **提示工程化描述**：工具描述如同给新员工写说明——消除歧义、明确输入输出、暴露隐含上下文。微小改进即可带来显著效果，Claude Sonnet 3.5 在 SWE-bench 上的突破正源于此。

**结论：** 为智能体写工具需要从确定性思维转向非确定性思维。通过系统化的评估驱动方法，确保工具随智能体能力的提升而同步进化。
