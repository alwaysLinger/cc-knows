# Advanced Tool Use 简要提炼

Anthropic 发布了三项 Beta 功能，让 Claude 能够动态发现、学习和执行工具，推动 AI 智能体 (Agent) 从简单的函数调用迈向智能编排。

## 核心问题

随着智能体需要操作的工具有几十甚至上百个，传统工具调用模式暴露三大瓶颈：工具定义占用大量上下文窗口（50K-134K tokens）、中间结果污染上下文、以及 JSON Schema 无法传达使用惯例导致参数错误。

## 三项功能

**1. Tool Search Tool（工具搜索工具）**：不再预先加载所有工具定义，而是按需搜索发现。Claude 仅加载当前任务所需的 3-5 个工具，将上下文占用从约 77K 降至 8.7K tokens（减少 85%），同时工具选择准确率显著提升（Opus 4 从 49% 提升至 74%）。

**2. Programmatic Tool Calling（编程式工具调用）**：Claude 通过编写 Python 代码来编排多个工具调用，而非逐次推理。中间结果在代码执行沙箱中处理，仅最终输出返回模型上下文。Token 消耗降低 37%，并减少多次推理延迟，准确率亦有提升。

**3. Tool Use Examples（工具使用示例）**：在工具定义中提供具体调用示例，让 Claude 学习格式惯例、参数组合和使用模式。测试显示准确率从 72% 提升至 90%。

## 核心结论

三者互补：Tool Search Tool 解决"找到对的工具"，Programmatic Tool Calling 解决"高效执行工具"，Tool Use Examples 解决"正确调用工具"。最佳实践是从最大瓶颈入手，逐步叠加。这些功能标志着工具使用从简单函数调用向智能编排的关键转变。
