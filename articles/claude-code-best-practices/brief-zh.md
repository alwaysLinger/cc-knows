# Claude Code 最佳实践：简要提炼

## 核心约束

Claude Code 的一切最佳实践都基于一个核心约束：**上下文窗口会快速填满，且性能随填充而下降。** 上下文窗口容纳整个对话历史、文件内容和命令输出，一次调试会话就可能消耗数万 token。当上下文接近饱和时，Claude 会"遗忘"早期指令、犯更多错误。管理上下文窗口是最重要的资源管理任务。

## 最高杠杆策略：给 Claude 验证自身工作的方式

提供测试、截图或预期输出，让 Claude 能自行检验结果。没有明确的成功标准，Claude 可能产出看似正确但实际不工作的代码，而你成为唯一的反馈环。

## 关键实践

1. **先探索，再规划，再编码** — 用 Plan 模式将研究、规划与实现分离，避免解决错误的问题。小任务可跳过规划。
2. **提供具体上下文** — 引用具体文件、指向代码模式、描述症状和期望结果。模糊指令导致反复修正。
3. **配置好环境** — 写精简的 CLAUDE.md（只放 Claude 无法自行推断的规则）；配置权限减少中断；安装 CLI 工具（如 `gh`）；连接 MCP 服务器；用 Hooks 做必须执行的动作；用 Skills 存放领域知识；用子代理处理隔离任务。
4. **有效沟通** — 像问资深工程师一样提问；对大型功能先让 Claude 采访你以明确需求。
5. **管理会话** — 频繁用 `/clear` 重置不相关的上下文；两次修正失败后清空重来；用子代理做调研以保护主对话上下文；善用检查点和会话恢复。
6. **自动化与扩展** — 用非交互模式集成 CI；并行运行多个会话（Worktree、桌面应用、Agent Teams）；批量任务用 `claude -p` 循环处理；Auto 模式实现无人值守执行。

## 常见失败模式

- 大杂烩会话（不相关任务混在一起）
- 反复修正同一问题（上下文被污染）
- CLAUDE.md 过长（重要规则被淹没）
- 信任但未验证（缺少测试/校验）
- 无边界探索（Claude 读取大量无关文件）

## 结论

这些模式是起点而非教条。根据具体情况判断何时规划、何时探索、何时清空上下文。留意什么有效、什么无效，逐步发展出适合自己的直觉。
