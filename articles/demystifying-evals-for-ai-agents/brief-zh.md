# AI Agent 评估解密 — 简要提炼

**原文**: [Demystifying Evals for AI Agents](https://www.anthropic.com/engineering/demystifying-evals-for-ai-agents)（Anthropic, 2026-01-09）

## 核心观点

AI Agent 的自主性、智能性和灵活性使其难以评估，但系统化的评估（eval）是可靠交付 Agent 的基石。没有 eval，团队只能被动响应生产问题；有了 eval，问题在影响用户前就能被发现，且其价值随时间复利增长。

## 主要论据

1. **评估结构**：评估由任务（task）、试验（trial）、评分器（grader）、轨迹（transcript）和结果（outcome）组成。Agent 评估比单轮评估复杂得多——错误会跨轮次传播和叠加，前沿模型甚至可能找到超越静态评估预设的创造性解法。

2. **三类评分器**：代码评分器（快速客观但脆弱）、模型评分器（灵活有弹性但需校准）和人工评分器（金标准但昂贵缓慢）。实践中应组合使用。

3. **能力评估 vs 回归评估**：能力评估探索"Agent 能做什么"，从低通过率起步；回归评估确保"Agent 还能做以前能做的事"，应接近 100% 通过率。高通过率的能力评估可"毕业"为回归评估。

4. **不同 Agent 类型的评估策略**：编码 Agent 侧重确定性测试；对话 Agent 需要多维度评分和模拟用户；研究 Agent 面对主观性和真值漂移的挑战；计算机使用 Agent 需在真实或沙盒环境中验证最终结果。

5. **非确定性处理**：用 pass@k 衡量"至少一次成功"的概率，用 pass^k 衡量"全部成功"的一致性。

6. **从零到一**：尽早收集 20-50 个来自真实失败的任务；设计无歧义的任务和参考解；构建稳定环境的评估框架；长期维护评估集健康度。

## 结论

评估是 Agent 开发的核心组件，而非可选附加。没有评估的团队陷入被动循环，投入评估的团队则将失败转化为测试用例、加速迭代。关键原则：尽早开始、任务真实、标准明确、评分器审慎设计、持续迭代。
