# 设计抗 AI 的技术评估

## 详细摘要

### 背景与核心问题

Anthropic 性能优化团队负责人 Tristan Hume 撰文，分享了团队在技术评估中与 Claude 模型反复较量的三轮迭代经历。核心问题是：随着 AI 能力的提升，传统技术评估的有效性正在迅速衰减——一个今天还能区分人类技能水平的测试，明天可能就被模型轻松解决。

### 测试的起源

2023 年 11 月，Anthropic 正在为 Claude 3 Opus 的训练做准备，获得了新的 TPU、GPU 和 Trainium 集群，但缺乏足够的性能工程师。作者在 Twitter 发帖招聘，引来了大量候选人，标准面试流程无法承载。于是他花了两周设计了一个线上编程测试，目标包括：

- **更长的时间窗口**：4 小时（后缩至 2 小时），更贴近真实工作节奏
- **真实环境**：无人监考，使用自己的编辑器
- **理解与工具构建空间**：性能优化需要理解系统并构建调试工具
- **兼容 AI 辅助**：明确允许使用 AI 工具，因为更长时限的问题 AI 更难完全解决
- **高信号量**：避免依赖单一洞见，提供多次展示能力的机会
- **无需特定领域知识**、**有趣味性**

他构建了一个 Python 模拟器，模拟类 TPU 加速器，包含手动管理的暂存内存（scratchpad）、VLIW（超长指令字）、SIMD（单指令多数据）和多核特性。任务是并行树遍历优化，候选人从串行实现起步，逐步利用机器的并行性。还包含一个初始调试 bug，考察工具构建能力。

### 早期成果

该测试成效显著：约 1000 名候选人完成测试，帮助招聘了数十名性能工程师，包括搭建 Trainium 集群和发布 Claude 3 Opus 后所有模型的工程师。测试对纸面经验不足的候选人尤其有价值——几名最优秀的工程师直接来自本科。反馈积极，许多候选人因享受过程而超时工作，最强的不限时提交包含了完整的优化迷你编译器。

### Claude Opus 4 攻克第一版

到 2025 年 5 月，Claude 3.7 Sonnet 已达到让超 50% 候选人不如直接委托 Claude Code 的水平。Claude Opus 4 更是在 4 小时限内提出了比几乎所有人类都更优的方案。作者的应对策略是：用 Claude 找到其自身开始挣扎的节点，作为第二版的新起点；编写更清晰的起始代码；新增更深的机器特性；移除已被 Claude 解决的多核部分（仅拖慢开发循环）；并将时限从 4 小时缩至 2 小时。

### Claude Opus 4.5 攻克第二版

Claude Opus 4.5 在 2 小时内逐步优化：解决了初始瓶颈、实现了常见微优化、不到 1 小时即达通过线。随后它像大多数人类一样，认为自己碰到了不可逾越的内存带宽瓶颈。但当被告知可达的周期数后，它思考片刻便找到了绕过瓶颈的技巧，继续调试和优化，2 小时时成绩追平了最高水平人类（而那人大量使用了 Claude 4 辅助）。在内部测试时计算（test-time compute）环境中，Claude 确认可以在 2 小时内超越人类，并随时间继续提升。

### 替代方案的考量

**禁止 AI 辅助**：被否决。不仅执行困难，而且作者认为应找到人类在有 AI 的环境中（如同实际工作）也能展现价值的方式。

**抬高门槛至超越 Claude**：同样被否决。Claude 工作速度快，人类通常花一半时间理解问题，指导 Claude 的人可能始终处于追赶状态，最优策略变成了旁观。

作者也担忧：设计新测试要么被 Claude 4.5 再次解决，要么难度过高到人类无法在 2 小时内完成。

### 尝试 1：换一个优化问题

作者选择了 TPU 寄存器转置与避免 bank 冲突的问题，让 Claude 一天内实现了修改。但 Claude Opus 4.5 发现了一个作者没想到的优化——转置整个计算而非数据。作者修补问题后，Claude 仍无法找到最优解。然而使用 Claude Code 的"ultrathink"功能和更长思考预算后，Claude 解决了它。问题在于：数据转置和 bank 冲突是跨平台经典问题，Claude 有大量训练数据可借鉴。

### 尝试 2：走偏门

作者需要人类推理能力能胜过 Claude 经验库的问题——足够"分布外"（out of distribution）。灵感来自 Zachtronics 游戏：使用高度受限的指令集，要求以最少指令数求解谜题。刻意不提供可视化和调试工具，构建调试工具本身就是测试的一部分——你可以精心插入打印语句，也可以让编程模型几分钟内生成交互式调试器。

测试结果：Claude Opus 4.5 失败了。同事验证确认人类可以超越 Claude。早期结果显示分数与候选人实际能力正相关。作者坦言放弃了原始测试的真实感和深度变化，但"真实性可能已是我们无法负担的奢侈品——原始测试因模拟真实工作而有效，替代测试因模拟新颖工作而有效"。

### 开放挑战

原始测试已发布在 GitHub，供任何人在不限时间条件下尝试。人类专家在足够长的时间范围内仍优于当前模型。基准成绩（模拟机器时钟周期数）：

- **2164 周期**：Claude Opus 4，测试时计算环境中数小时
- **1790 周期**：Claude Opus 4.5，常规 Claude Code 会话
- **1579 周期**：Claude Opus 4.5，2 小时测试时计算
- **1548 周期**：Claude Sonnet 4.5，超 2 小时测试时计算
- **1487 周期**：Claude Opus 4.5，11.5 小时测试时计算
- **1363 周期**：Claude Opus 4.5，改进版测试时计算环境数小时

突破 1487 周期（发布时 Claude 最佳成绩）者可发送代码和简历至 performance-recruiting@anthropic.com，或通过 Anthropic 常规流程申请（使用当前的抗 Claude 测试）。
