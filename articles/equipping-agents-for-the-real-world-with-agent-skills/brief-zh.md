# 为智能体装备现实世界能力：Agent Skills

## 简要提炼

Anthropic 推出了 **Agent Skills**——一种通过文件和文件夹为通用智能体（Agent）注入领域专业能力的新机制。

### 核心问题

尽管 Claude 等大模型已能操作完整的计算环境（如 Claude Code 可执行代码、访问文件系统），但它们缺乏特定领域的**程序性知识**和组织上下文。为每个场景定制独立智能体成本高昂、难以复用，亟需一种可组合、可扩展、可移植的方案。

### 核心方案：Agent Skills

Skill 的本质是一个包含 `SKILL.md` 的目录，其中收纳了指令、脚本和资源。其设计遵循**渐进式披露（Progressive Disclosure）**原则，分三层加载信息：

1. **元数据层**：启动时仅将 `name` 和 `description` 加载到系统提示，让 Claude 知道何时该使用该 Skill；
2. **核心指令层**：当任务相关时，Claude 读取完整 `SKILL.md`；
3. **附加文件层**：复杂场景下可引用子文件（如 `forms.md`），按需加载，避免上下文膨胀。

这种设计使 Skill 的上下文容量**实际上无上限**，因为只有必要信息才被载入上下文窗口。

### 关键特性

- **代码执行**：Skill 可包含脚本，Claude 可直接运行而非将代码读入上下文，兼顾效率与确定性；
- **跨平台支持**：已在 Claude.ai、Claude Code、Agent SDK 和开发者平台上线，并已发布为开放标准（agentskills.io）。

### 开发与安全建议

- 先评估智能体不足之处，再逐步构建 Skill；
- 文件过大时拆分为子文件，互斥上下文分开存放；
- 从 Claude 视角观察其使用模式并迭代；
- 仅安装可信来源的 Skill，审计代码依赖和网络连接。

### 愿景

未来将支持 Skill 的全生命周期管理（创建、发现、分享），探索与 MCP 服务器的互补，并让智能体自主创建和评估 Skill，将行为模式固化为可复用能力。
