# 为智能体装备现实世界能力：Agent Skills

## 详细摘要

### 背景与动机

随着模型能力提升，我们已经可以构建与完整计算环境交互的通用智能体（Agent）。Claude Code 就是典型例子——它能跨领域完成复杂任务，拥有本地代码执行和文件系统访问能力。然而，通用智能体在特定领域缺乏**程序性知识（Procedural Knowledge）**和组织上下文（Organizational Context）。为每个用例单独构建定制智能体则碎片化且不可扩展。这一矛盾催生了 **Agent Skills**——一种以文件和文件夹组织指令、脚本与资源，让智能体动态发现和加载的方案。

### Skill 的定义与类比

构建一个 Skill 类似于为新员工编写入职指南：你将流程性知识捕获并打包成可组合的资源，供 Claude 使用，从而将通用智能体转变为符合特定需求的专业智能体，而无需为每个场景从头造轮子。

### Skill 的结构剖析

以 Anthropic 的 **PDF Skill** 为例。Claude 天然理解 PDF 内容，但直接操作 PDF（如填写表单）能力有限。该 Skill 为 Claude 赋予了这些新能力。

Skill 的最简形式是一个包含 `SKILL.md` 的目录。该文件必须以 YAML frontmatter 开头，包含 `name` 和 `description` 两项必需元数据。启动时，智能体将所有已安装 Skill 的名称和描述预加载到系统提示中。

**渐进式披露（Progressive Disclosure）** 是 Skill 的核心设计原则，分三层工作：

1. **第一层——元数据**：仅加载 `name` 和 `description`，让 Claude 判断何时触发该 Skill，无需加载全部内容；
2. **第二层——核心指令**：当 Claude 判定 Skill 与当前任务相关时，读取完整 `SKILL.md` 到上下文；
3. **第三层及以上——附加文件**：Skill 可在目录中打包额外文件，由 `SKILL.md` 按名称引用。Claude 仅在需要时才导航和发现这些文件。

在 PDF Skill 示例中，`SKILL.md` 引用了两个附加文件（`reference.md` 和 `forms.md`）。将表单填写指令分离到 `forms.md`，保持了核心文件的精简——Claude 仅在填写表单时才读取它。

这种设计的核心优势：由于拥有文件系统和代码执行工具的智能体无需一次性读取整个 Skill，Skill 可捆绑的上下文量**实际上无上限**。

### 上下文窗口中的 Skill 触发流程

1. 上下文窗口包含核心系统提示 + 所有已安装 Skill 的元数据 + 用户消息；
2. Claude 判断需要 PDF Skill，通过 Bash 工具读取 `pdf/SKILL.md`；
3. Claude 进一步读取 `forms.md` 等附加文件；
4. 加载完相关指令后，Claude 执行用户任务。

### Skill 与代码执行

大语言模型在许多任务上表现出色，但某些操作更适合传统代码：例如排序列表通过 token 生成远比运行排序算法低效，许多应用也需要代码才能提供的确定性可靠性。PDF Skill 内含一个预编写 Python 脚本，可读取 PDF 并提取所有表单字段。Claude 无需将脚本或 PDF 加载到上下文即可运行，且由于代码是确定性的，该工作流一致且可重复。

代码在 Skill 中可同时充当**可执行工具**和**文档参考**，作者应明确区分二者。

### 开发与评估指南

- **从评估出发**：在代表性任务上运行智能体，观察其不足之处，然后增量式构建 Skill 弥补缺陷；
- **为规模而结构化**：当 `SKILL.md` 变得臃肿时拆分为子文件并引用；互斥或极少共用的上下文分开存放以减少 token 消耗；区分代码是"直接运行"还是"读入上下文作为参考"；
- **站在 Claude 视角思考**：监控 Claude 在真实场景中如何使用 Skill，关注意外轨迹或对某些上下文的过度依赖；特别注意 `name` 和 `description` 的措辞，因为 Claude 依据它们决定是否触发 Skill；
- **与 Claude 协作迭代**：在工作过程中让 Claude 将成功方法和常见错误捕获为 Skill 中的可复用上下文和代码；若 Skill 使用偏离预期，让 Claude 自我反思原因。这帮助你发现 Claude 真正需要的上下文，而非事先猜测。

### 安全考量

Skill 通过指令和代码赋予 Claude 新能力，但也意味着恶意 Skill 可能引入漏洞或诱导 Claude 泄露数据、执行非预期操作。建议：仅安装来自可信来源的 Skill；对不可信来源的 Skill 进行彻底审计——检查所有捆绑文件、代码依赖、图像和脚本，特别关注指示 Claude 连接外部网络资源的指令或代码。

### 未来展望

Agent Skills 已在 Claude.ai、Claude Code、Agent SDK 和开发者平台上线，并已发布为开放标准（agentskills.io）。未来将支持 Skill 全生命周期（创建、编辑、发现、分享、使用）；探索 Skill 与 MCP（Model Context Protocol）服务器的互补——教授智能体涉及外部工具的更复杂工作流；更长远地，让智能体自主创建、编辑和评估 Skill，将行为模式固化为可复用能力。

Skill 的核心理念是简单：简单的概念，简单的格式。这种简使得组织、开发者和终端用户都能更轻松地构建定制智能体并赋予其新能力。
